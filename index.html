📘 ビザンチン合意形成と裏切り者排除の仕組み

ビザンチン障害（Byzantine Fault）とは、意図的に偽のメッセージや異なる投票（「⚔️ 攻撃」vs「🛡️ 撤退」の分裂投票）を送信してシステムを混乱させようとするノードが含まれる障害です。

N ≥ 3f + 1

裏切り者が排除される仕組み：
正常なノード群は、受け取ったメッセージのうち正当な署名とダイジェストを持つものだけを集計します。集まった票数がクオラム閾値 2f + 1 に達しない偽の提案（裏切りノードの「🛡️ 撤退」意見）は「無効（REJECTED）」として即座に破棄されます。裏切り者が $f$ 台以下であれば、正常ノードの「⚔️ 攻撃」票のみで必ず $2f + 1$ の合意が成立します。

PBFTの3フェーズ手順

1

Pre-Prepare フェーズ

リーダー（Primary）が「⚔️ 攻撃」の提案を全ノードへ送信します。

2

Prepare フェーズ (裏切り者の検証 & 排除)

全ノード間でお互いに投票（Prepareメッセージ）を送信。正常ノードは裏切り者の「🛡️ 撤退」票を破棄し、2f + 1 個の整合「⚔️ 攻撃」メッセージが集まった提案のみを承認（PREPARED）します。

3

Commit フェーズ

全ノード間でCommitメッセージを交わし、2f + 1 票に達した「⚔️ 攻撃」提案を最終確定（COMMITTED）してクライアントに実行結果を返信します。

🚀 GitHub Pages での本アプリ公開手順

このアプリケーションはピュア HTML/CSS/JavaScript で構成されており、ビルド不要で GitHub Pages に直接公開できます。

手順 1: GitHub にリポジトリを作成してプッシュ

git init git add . git commit -m "Update BFT Traitor Exclusion Simulator" git branch -M main git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPOSITORY.git git push -u origin main

手順 2: GitHub Pages の有効化

  1. GitHub のリポジトリページを開き、Settings タブをクリックします。
  2. 左サイドメニューの Pages を選択します。
  3. Build and deployment の Source で Deploy from a branch を選択します。
  4. Branch を main / / (root) に設定して Save をクリックします。

手順 3: アクセス確認

数分後に https://YOUR_USERNAME.github.io/YOUR_REPOSITORY/ でアプリが全世界に無料公開されます！受講生にこのURLを共有して学習に活用してください。
